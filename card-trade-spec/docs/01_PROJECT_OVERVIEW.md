# 01 - 要件定義書

## 1. プロジェクト概要

### 1.1 事業概要
- **事業形態**: 法人
- **事業内容**: トレーディングカード・デジタルカードの買取および販売
- **古物商許可**: 取得予定（サイトURLを届出に使用）

### 1.2 サイトの目的
1. 古物商許可の届出用公式URLとして機能すること
2. トレーディングカードの買取受付（オンライン査定含む）
3. カードのEC販売
4. 事業の信頼性・ブランドの訴求

### 1.3 取扱商品カテゴリ

| カテゴリ | 種別 | 備考 |
|----------|------|------|
| ポケモンカード | 実物TCG | メインカテゴリ |
| 遊戯王OCG | 実物TCG | |
| ワンピースカード | 実物TCG | |
| その他TCG/トレカ | 実物TCG | ドラゴンボール、ヴァイスシュヴァルツ等 |
| デジタルカード | NFT/デジタル | ガチャサイト発行のカード等 |

## 2. 機能要件

### 2.1 公開サイト
- トップページ（ヒーロー、カテゴリ紹介、特徴、新着商品、お知らせ）
- 買取価格一覧（カテゴリ別タブ、検索、テーブル）
- 買取の流れ（4ステップ図解、買取方法3種、FAQ）
- オンライン査定フォーム（画像アップロード対応）
- ECショップ（商品一覧・詳細・カート・注文フォーム）
- 会社概要（法人情報、古物商許可情報）
- 特定商取引法に基づく表記
- お問い合わせ
- お知らせ
- プライバシーポリシー

### 2.2 管理画面
- ダッシュボード（KPI表示）
- 商品管理（CRUD + 画像アップロード）
- 買取価格管理（インライン編集 + CSVインポート）
- 査定管理（ステータス管理 + メール送信）
- 注文管理（ステータス管理）
- お知らせ管理（Markdown対応）

### 2.3 認証
- Supabase Auth（メール+パスワード）
- adminロールのみ管理画面アクセス可

## 3. 非機能要件
- Lighthouse Performance 90以上
- モバイルファースト・レスポンシブ
- Supabase RLS必須
- SEO（メタタグ、OGP、sitemap、構造化データ）
- アクセシビリティ（セマンティックHTML、キーボードナビ）

## 4. Phase 1 スコープ外
- オンライン決済（Stripe等）
- 多言語対応
- チャットボット
- ポイント/会員ランク
- SNSログイン
- プッシュ通知
